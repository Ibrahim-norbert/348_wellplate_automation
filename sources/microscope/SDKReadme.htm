Software improvements and new functionality added to Andor SDK 2.103.30022.0

 

New Features: 

1.      SetSizeOfCircularBufferMegaBytes so that user could change circular buffer size.

2.      Added reading/writing IRIG data to FITS files

3.      Updated FTDI library,

4.      Added new functionality to the ShamrockSDK to control the iris opening widths on the new spectrographs that support these new hardware controls. See the ShamrockSDK help file for more details.

5.      Re-named the ShamrockSDK to ATSpectrographSDK because the new line of spectrographs are called Kymera and we wanted to ensure there was no confusion over whether or not the SDK supported them. The new ATSpectrographSDK is completely backward compatible with the old ShamrockSDK so any old code will just need recompiled and relinked against the new headers/libraries with no code changes required. Any new projects should use the new ATSpectrographSDK API.

      Changes from the old ShamrockSDK to the new SpectrographSDK are mainly naming and parameter types. For example the old ShamrockInitialize function has been replaced with ATSpectrographInitialize which is essentially the same function, only renamed. Parameters that were previously an int, such as the error code passed to the ShamrockGetFunctionReturnDescription have been replaced with enum types for the ATSpectrographGetFunctionReturnDescription.

      The *DetectorOffset function group has been largely simplified with the old ShamrockSetDetectorOffset, ShamrockGetDetectorOffset, ShamrockSetDetectorOffsetPort2 and ShamrockGetDetectorOffsetPort2 functions having no direct replacement with the new API and the ShamrockSetDetectorOffsetEx and ShamrockGetDetectorOffsetEx functions being replaced with ATSpectrographSetDetectorOffset and ATSpectrographGetDetectorOffset respectively.

      For a full listing of the new API please consult the atspectrograph.chm help file or read the atspectrogtaph.h header file for the SDK. The intent of the new changes was both to avoid confusion and to simplify the API.

 

Bug Fixes:

1.      Fixed issue with signal level in FVB mode when switching horizontal shift speeds on Ultra 888.

2.      Fixed issue with time stamp in different time zones.

3.      Fixed IRIG-B support on iKon-LR.

4.      Fixed control of SR303i through I2C on Linux.

5.      Fixed issue causing different intensities on Ultra when re-initialized.

6.      Fixed issue in the spectrograph SDK so that it only looks for Andor spectrographs.

7.      Fixed issue of incorrect FIFO full error on Newton.

8.      Fixed issue of GetGrating reporting wrong grating.

9.      Fixed issue of extra fire pulse seen in FVB mode in a single scan.

10.  Fixed issue where beam tilt error occurred and message box appeared.

11.  Fixed reported amplifier, pre-amp gain and shuttering mode in FITS file.

12.  Fixed issue with number of frames being limited to 8192 whenever DDG + IOC + Ext Trigger are used together.

13.  SetHSSpeed now updates bits per pixel to valid value for iKon-XL.

14.  Fixed issue with setting certain exposure times in FVB mode on iVac 316.

15.  Fixed crash when trying to save as fits from Linux.
